Identification of hosts behind a NAT device utilizing multiple fields of IP and TCP.（流量分析与预处理方法、无监督学习与聚类技术）
方法：这篇文章采用 被动流量监测 方式，通过测量 TCP 流的 RTT 来收集和分析流量特征，避免了传统基于协议字段（如 TTL、UA 等）的检测方法。在数据预处理阶段，文献对 TCP 流 RTT 进行去噪处理，使用 Mean Shift 聚类算法（MSCR）进行无监督学习，识别 Tethering 设备的行为。
解决问题：文章提出了一种基于 RTT 测量 的新架构，用于有效识别 Tethering 行为。该方法通过 被动监测网络流量，实现了对 Tethering 行为的准确检测，准确率达到 97.50%，并在移动互联网环境中表现良好。
细粒度科学问题：文章解决了 TCP 流 RTT 在 Tethering 行为检测中的应用问题，尤其是在 加密与字段篡改情况下的鲁棒性。它还探讨了如何 去噪 和 处理 RTT 异常波动，减少由于设备性能差异（如转发性能差）导致的误判，确保检测模型在不同设备条件下的稳定性。
不足：文章依赖 TCP 流 RTT 特性进行检测，但在某些环境下（如无线网络中设备的高延迟抖动），RTT 可能不稳定，影响检测结果的准确性。此外，无监督学习模型的泛化性仍需在更大规模、更多设备的真实环境中验证，当前实验数据集较为有限，且测试集与真实环境存在差距，可能导致外推性不足。

A novel approach for detecting vulnerable IoT devices connected behind a home NAT.（设备类型的识别，都是NAT后设备的识别，但是我们是数量，这篇文章是设别的类型）
方法：该文献提出了一种基于 NetFlow 的流量分析方法，通过收集 流量元数据（如源端口、目标端口、协议、字节数等）来检测 家庭网络中的易受攻击的 IoT 设备。该方法使用 机器学习分类算法（如 LGBM）训练分类器，以识别家庭 NAT 后面连接的易受攻击设备模型。该方法避免了传统的 深度包检测（DPI），而是聚焦于 流级特征，并且能够处理 加密流量、非 TCP 或非 DNS 流量。
解决问题：文献提出的方法旨在解决 IoT 设备在家庭网络中被 NAT 隐藏 的问题，帮助 电信运营商（telco） 在 IoT 设备被恶意软件利用之前识别出易受攻击的设备，防止 DDoS 攻击对运营商基础设施造成影响。
细粒度科学问题：1.NAT 后设备识别问题：当多个设备通过同一个NAT网关共享公网 IP 时，如何准确区分每个设备的流量？是否可以仅依赖流量元数据，在不依赖 IP、端口和应用层信息的情况下，完成设备的精准识别？ 2. 加密流量与流量分析：在面对 加密流量 时，如何从 NetFlow 数据中提取有效特征来识别 易受攻击的 IoT 设备，而不依赖传统的应用层数据（如 DNS 请求和 HTTP 头部信息）？
不足：1. 该方法依赖于 NetFlow 数据，而 NetFlow 只能提供 流量的元数据，并不能完整分析流量中的 payload，因此其应用受到一定限制，特别是在设备进行加密通信时。2. 该方法的 训练数据 和 评估数据 主要来源于实验室收集的家庭 IoT 设备流量，这些数据可能无法完全代表 全球范围内 真实用户的 多样化 IoT 设备，因此泛化能力需进一步验证。3. 对于 设备固件更新 或 漏洞修复的IoT设备，现有的分类器可能会失效，必须定期进行 重新训练 以适应新版本或新设备。

Identification and Counting of Hosts Behind NAT Using Machine Learning.
方法：文章提出了一种基于 机器学习（ML） 的方法来 识别和计数 NAT 后的设备。方法包括：数据收集：使用 Wireshark 捕获网络流量，采用 Tranalyzer 工具提取流量的 统计特征（如 TCP ACK、源端口、窗口大小等）。特征选择：通过 过滤法 和 包装法（Wrapper Method） 选择最具代表性的 14 个特征。机器学习算法：使用 决策树（DT） 和 随机森林（RF） 算法进行多类分类，评估流量是否来自单个或多个主机。
解决问题：文献的核心是 计数 NAT 后的设备数量，并比较了 基于签名的方法 和 基于机器学习的方法，提出了一个不依赖于 IP 地址和端口号的 机器学习方法，通过分析 流量统计特征 来推测 NAT 后的设备数量，并准确区分单主机与多主机的流量。比较结果显示，机器学习方法（DT）比传统的签名方法具有更高的 准确率（约 93%），尤其是在恶意流量数据集上，能够识别并计数 恶意主机。
细粒度科学问题：1.NAT 隐匿设备识别：如何通过 网络流量的统计特征 来 识别 NAT 隐匿的多个设备，而不依赖于 IP 地址 或 端口号？2.恶意流量的设备计数：如何在恶意流量中区分 单一主机 与 多个主机 发起的攻击，确保 准确计数？3.特征选择的有效性：在 不同网络环境 中，如何选择与 用户行为 和 网络环境无关的特征，确保模型的 高效性与鲁棒性？
不足：计算复杂度：尽管 决策树（DT） 算法具有较快的训练时间（18秒），但 随机森林（RF） 的训练时间较长（182秒），且 性能提升仅为 0.1%，因此选择 DT 更为高效，但 RF 在某些情况下可能表现稍好。跨数据集的泛化能力：虽然模型在 ICS 数据集 上表现出色（准确率 98%），但 LAB 数据集 的表现略有下降，表明模型 训练集与测试集不同 时，性能可能受到影响。恶意流量检测的局限性：虽然方法能够有效 识别恶意主机，但对于 多种 NAT 行为 和 多类设备，模型仍需在 更广泛的网络环境中 进行验证。

解决问题：本研究的核心问题是 如何识别和量化 NAT 后的 IoT bot，尤其是被 Mirai 恶意软件感染的设备。通过该方法，可以识别并量化 Internet-scale IoT botnet，并准确地为每个 NAT 地址 推断出感染的 IoT 设备数。文章展示了该方法在 恶意流量数据上的准确性，达到了 93% 的准确率，超越了现有的浅层学习方法。
